## 📝 测试步骤

### Step 1: 进入文章编辑页面
1. 登录后台管理系统
2. 左侧菜单：`文章资讯` → `文章管理`
3. 点击顶部 `添加` 按钮

### Step 2: 打开Markdown粘贴对话框
1. 在编辑器顶部找到蓝色的 **「粘贴Markdown」** 按钮
2. 点击按钮
3. 应该弹出一个对话框

### Step 3: 粘贴测试内容

将以下内容复制粘贴到对话框：

```markdown
# 测试标题

这是一段**加粗文本**和*斜体文本*的示例。

## 功能列表

- 支持标题
- 支持**加粗**
- 支持*斜体*
- 支持列表

### 代码示例

\`\`\`javascript
function hello() {
    console.log('Hello World');
}
\`\`\`

> 这是一段引用文本

[访问百度](https://www.baidu.com)

---

*测试完成*
```

### Step 4: 转换并验证
1. 点击 **「转换为HTML」** 按钮
2. 等待1-2秒
3. 应该显示绿色提示：**「Markdown内容已成功转换为HTML」**
4. 对话框自动关闭
5. 检查编辑器中的内容

### Step 5: 检查结果

在编辑器中应该看到：
- ✅ 大标题："测试标题"（大字体）
- ✅ "加粗文本"显示为粗体
- ✅ "斜体文本"显示为斜体
- ✅ 列表有正确的缩进和符号
- ✅ 代码块有等宽字体和灰色背景
- ✅ 引用有左侧竖线和背景色
- ✅ 链接可点击（蓝色下划线）
- ✅ 底部文字显示为斜体

## ❌ 如果还是报错

### 检查清单

1. **刷新浏览器**
   - 按 `Ctrl+Shift+R` (Windows)
   - 按 `Cmd+Shift+R` (Mac)
   - 强制刷新缓存

2. **清除浏览器缓存**
   - 打开开发者工具（F12）
   - 右键点击刷新按钮
   - 选择"清空缓存并硬性重新加载"

3. **检查控制台错误**
   - 打开开发者工具（F12）
   - 切换到 Console 标签
   - 查看是否有错误信息
   - 截图发送给技术支持

4. **确认依赖已安装**
   ```bash
   cd admin
   npm list marked
   ```
   应该显示：`marked@x.x.x`

5. **重新构建**
   ```bash
   cd admin
   rm -rf node_modules dist
   npm install
   npm run build
   ```

## 💡 使用技巧

### 技巧1：从其他平台复制
- 知乎文章：复制内容 → 粘贴到对话框 → 转换
- CSDN博客：复制Markdown源码 → 粘贴 → 转换
- 简书文章：复制内容 → 粘贴 → 转换

### 技巧2：分段处理长文档
- 长文章建议分章节转换
- 每次转换一个章节
- 避免一次性处理太多内容

### 技巧3：结合手动编辑
- 转换后可以继续手动调整
- 在编辑器中添加图片
- 调整格式和样式
