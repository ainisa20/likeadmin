## 测试步骤

### 步骤1：登录后台
1. 访问：`http://localhost:24678` 或 `http://localhost:18088/admin`
2. 登录账号：admin / 123456

### 步骤2：进入文章编辑页面
1. 点击左侧菜单：`文章资讯` → `文章管理`
2. 点击顶部 `添加` 按钮

### 步骤3：测试Markdown粘贴功能
1. 在编辑器顶部找到「**粘贴Markdown**」按钮（蓝色按钮）
2. 点击按钮，应该弹出对话框
3. 将上述测试数据粘贴到文本框
4. 点击「**转换为HTML**」按钮
5. 等待转换完成（应该显示"转换成功"提示）
6. 检查编辑器中的内容是否正确显示

### 步骤4：验证转换效果
检查以下格式是否正确：
- [ ] 标题（H1-H6）有正确的字体大小
- [ ] 加粗文本显示为粗体
- [ ] 斜体文本显示为斜体
- [ ] 列表有正确的缩进和符号
- [ ] 引用有灰色的背景和左边框
- [ ] 代码块有等宽字体
- [ ] 链接可点击且有下划线
- [ ] 表格有边框和对齐
- [ ] 分割线显示正确

### 步骤5：保存并预览
1. 填写文章标题（如：Markdown测试文章）
2. 选择文章分类
3. 点击 `保存` 按钮
4. 保存后访问PC端预览：
   - 地址：`http://localhost:18088/pc/information/detail/{文章ID}`
   - 检查前端渲染是否正常

## 常见问题排查

### 问题1：点击"粘贴Markdown"按钮没反应
**原因**：可能是marked库未正确安装
**解决**：
```bash
cd admin
npm install marked --save
```

### 问题2：转换后内容为空
**原因**：可能是编辑器API调用问题
**解决**：打开浏览器控制台查看错误信息

### 问题3：转换失败提示错误
**原因**：Markdown格式可能有误
**解决**：检查并修正Markdown语法

### 问题4：前端显示HTML标签
**原因**：v-html指令可能有问题
**解决**：检查PC端 `news_detail.vue` 是否使用 `v-html`

### 大文档测试
使用以下命令生成大文档：
```bash
# 生成包含1000个段落的Markdown文档
for i in {1..1000}; do echo "## 段落 $i"; echo "这是第 $i 段测试内容"; echo ""; done > large-test.md
```

测试转换时间：
- < 1000字符：应该 < 1秒
- 1000-10000字符：应该 < 3秒
- > 10000字符：可能需要3-5秒
